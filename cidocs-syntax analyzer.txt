syntax analyzer reads 3 bytes from fle generated by lexical analyzer (because every token is made from 3 bytes) and recognizes it , based on that it expects number of things after the token , and it goes from there
when it fnds something unexpected it gives an error 

new tokens can be generated from syntax analyzer :
IDTEQU will turn to VAR if there isnt "SB1" token after EQU
if found SB1 it turn IDTEQUSB1 to "ARD" 
when finds IDTSB1 it turns them to "ARU" 
when finds IDTLPR it turns them to "FCL" 
end of function call is "FCE"
SB2 in end of ARD turns to "ADE" 
SB2 in end of ARU turns to "ARE" 
end of while is "EWL" 
end of if is "EIF"
end of int() or str() is "2ND" 
end of moving bytes is RPR 
BR1 in FDF is BR1 
BR1 in while/if is "GOT"
BR2 in FDF/while/f is "END"


rules : 

newline / terminator :
BWD , registers ,string , int /float , print , function def , to string , to int , not , negate , minus/plus , IDT/VAR/FCL/ARD/ARU , if / while , moving bits , mobing bytes , system call , pointer , address of , newline/terminator ,LPR , pop/push ,end of file , return , BR2 ,true,flase null


VAR/ARD expect : registers , byte/word/dword ,true/false/null , minus/plus , integer/floating point int , string , not , LPR ,int() , str() ,negate , pointer , address of , bits and bytes moving , idt /ARU/FCL
and input ONLY for VAR

IDT/PTR expect : math operators , or , and ,comparison ,increase / decrease , terminator/newline 

expects comma in function call , functon def and array def 
expects RPR in function call , when inside LPR and in function def 
expects SB2 in ARU/ARD

AOD expects : comparison , or/and ,math operators , terminator/newline 

expects comma in function call , functon def and array def 
expects RPR in function call , when inside LPR and in function def 
expects SB2 in ARU/ARD

in pointer and address of , it first reads an id 
tokenz ex : AODIDTexample
this id is neccesary 


NEG expects : integer/float , byte/word/dword , registers ,not , true/false , IDT/FCL/ARU , pointer , int() 

math operators expect : registers , byte/word/dword , pointer , integer/float int , LPR , int() , not , IDT/FCL/ARU , true/false 

null expects : comparison , or/and , terminator/newline 

expects comma in Functin call and array def 
expects RPR in function call and when inside LPR 
expects SB2 in ARU/ARD 

case:LPRNUL 
expects comparison ,and/or because null = nothing and LPR should contain integer 
(!null) is acceptd

true/false: same as null but they expect math operators too 
and are allowed alone in LPR 
(false) is accepted 

string expects : exactly like null  

integer/floating pont integer expect : math operators , or/and , comparison , terminator/newline , increase/decrease 

expects comma in Functin call and array def 
expects RPR in function call and when inside LPR 
expects SB2 in ARU/ARD 

increase/decrease expect : comparison , math operators , and/or ,terminator/newline 

expects comma in Functin call and array def 
expects RPR in function call and when inside LPR 
expects SB2 in ARU/ARD 

LPR expects : regsters , byte/word/dword , integer/float int ,address of , pointer , IDT , function call , ARU , minus/plus ,LPR ,true/false/null ,string , int() , str() ,not ,negate ,moving bits and bytes 

comparison/not/and/or expect : same as lpr 

RPR expects : RPR , math operators , or/and , comparison ,termonator/newline  

expects comma in Functin call and array def 
expects RPR in function call and when inside LPR 
expects SB2 in ARU/ARD 

SB1/CMA : same as LPR 

SB2 : in case it is converted to ADE it expects termnator or newline only 
in case its converted to ARE it expects same as RPR  

print expects : LPR* : same as lpr
terminator/newline* 

input expects : terminator/newline*

FCL/ARD expect : same as lpr without the rules of cases 
f("hI") is accepted 

ARU : like LPR but doesnt expect str() and moving bits and bytes 

push expects :  LPR* :
 LPR, registers , integer/floating int , string , not , true/false/null , minus/plus , address of , pointer , FCL/IDT/ARU 
terminator/newline* 

pop expects : LPR* :
RPR , IDT , registers 
terminator/newline*

if/while expect : LPR* :
exactly as comparison 
RPR* 
while and if require statements to be inside {}
but allow terminator/newline before BR1 

func def (FNC) expects : IDT* : LPR* : 
IDT (expects either comma or rpr) 
comma in func def expects only IDT 
RPR* 

return expects : exactly like LPR but without moving bits and bytes 

basicall a function ends when it reaches return , which is needed in function def , so syntax allows you to define function as this :
func:x()
return 0;

or like this :

func:X(){
return 0;}

so after line of "func:" is over it expects either newline or terminator or BR1 
in case of newline or terminator it again expects a BR1 , if not found it decreases the file pointer by 6 (2 tokenz) and uses token of newline/terminator 
this way the function def will end after return and BR2 after return will cause an error 
but if found BR1 it will require BR2 after return


moving bits : INT* : DOT* 
same as comparison 

moving bytes : LPR* :
IDT , integer/floating int , string , pointer , registers 
(all expect comma only) 
comma expects only IDT or integer 
IDT and integer after comma expect only RPR 
and RPR expect only comparison or terminator/newline 

int() expects : LPR* :
string , IDT/FCL/ARU , pointer 
RPR* 

str() expects : LPR* :
integer/floating int , IDT/FCL/ARU , pointer , LPR 
RPR*
